# S3 Lab Service

Простое веб-приложение на FastAPI для работы с файлами в Yandex Object Storage (S3).

## Назначение проекта

Этот проект представляет собой минималистичный REST API сервис, который позволяет:

* Просматривать список файлов в заданном бакете S3.
* Загружать новые файлы в бакет.
* Удалять существующие файлы из бакета.
* Генерировать временные (presigned) ссылки для безопасного скачивания или просмотра файлов.

## Структура проекта

s3-lab/

## Установка зависимостей

1.  Убедитесь, что у вас установлен Python 3.7 или выше.
2.  Создайте виртуальное окружение (рекомендуется):
    ```bash
    python -m venv venv
    ```
3.  Активируйте виртуальное окружение:
    *   **Windows:** `venv\Scripts\activate`
4.  Установите зависимости из файла `requirements.txt`:
    ```bash
    pip install -r requirements.txt
    ```

## Конфигурация (файл .env)

Создайте файл `.env` в корне проекта (рядом с `main.py`) и заполните его своими данными. **НИКОГДА не коммитьте этот файл в систему контроля версий!**

**Пример файла `.env` (подставьте свои значения):**
```env
# Конечная точка Yandex Object Storage
S3_ENDPOINT_URL=https://storage.yandexcloud.net

# Имя вашего бакета в Yandex Cloud
S3_BUCKET_NAME=your-bucket-name

# Ключ доступа (получается в IAM сервисе Yandex Cloud)
AWS_ACCESS_KEY_ID=your_access_key_id

# Секретный ключ (получается в IAM сервисе Yandex Cloud)
AWS_SECRET_ACCESS_KEY=your_secret_access_key

# Регион бакета (например, ru-central1)
AWS_DEFAULT_REGION=ru-central1
```

## Запуск приложения

Запустите сервер приложения с помощью `uvicorn`:

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

*   `--reload`: Включает режим перезагрузки при изменении кода (удобно для разработки).
*   `--host 0.0.0.0`: Позволяет подключаться к серверу с других машин в сети.
*   `--port 8000`: Запускает сервер на порту 8000.

После запуска Swagger UI (интерактивная документация API) будет доступен по адресу `http://localhost:8000/docs`.

## Описание endpoint'ов

*   **`GET /files`**
    *   **Назначение:** Получить список всех файлов в бакете.
    *   **Ответ (успех):** JSON-массив с именами файлов.
    *   **Пример ответа:** `{"files": ["image.jpg", "document.pdf"]}`

*   **`POST /upload`**
    *   **Назначение:** Загрузить новый файл в бакет.
    *   **Тело запроса:** Файл, отправленный как `multipart/form-data`.
    *   **Ответ (успех):** JSON с сообщением об успехе.
    *   **Пример ответа:** `{"message": "Файл example.txt успешно загружен"}`

*   **`DELETE /files/{filename}`**
    *   **Назначение:** Удалить файл по его имени.
    *   **Параметры пути:** `filename` - имя файла для удаления.
    *   **Ответ (успех):** JSON с сообщением об успехе.
    *   **Пример ответа:** `{"message": "Файл old_file.txt успешно удалён"}`

*   **`GET /presign/{filename}`**
    *   **Назначение:** Сгенерировать временную ссылку для скачивания или просмотра файла. Ссылка действует ограниченное время (по умолчанию 1 час).
    *   **Параметры пути:** `filename` - имя файла для которого нужна ссылка.
    *   **Ответ (успех):** JSON с полем `presign_url`, содержащим саму ссылку.
    *   **Пример ответа:** `{"presign_url": "https://storage.yandexcloud.net/...?Expires=...&Signature=..."}`
